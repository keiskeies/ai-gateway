# Dashboard Layout Optimization Plan

## Problem

Dashboard 页面数据过多（7个统计卡片 + 8个虚拟大模型卡片 + 20个平台卡片），内容远超一屏。

## Solution: Compact Two-Column Layout (All Data Visible)

### 1. Stats Bar — 一行紧凑统计

把当前两行 7 个 StatCard 合并为一行 4 个紧凑项：

| 请求数 + 成功率 | 平均延迟 | 活跃平台 | Token 统计 |
|---|---|---|---|
| 合并为一个卡片 | 单独一个 | 合并显示 x/y | 合并输入/输出 |

用 `padding="sm"`，label 和 value 更紧凑。

### 2. Two-Column Detail — 左右并排

```
┌──────────────────────────┬──────────────────────────┐
│  虚拟大模型 (8)            │  已接入平台 (20)           │
│  2列网格 = 4行            │  4列网格 = 5行            │
│                          │                          │
│  [card] [card]           │  [card] [card] [card] [c]│
│  [card] [card]           │  [card] [card] [card] [c]│
│  [card] [card]           │  [card] [card] [card] [c]│
│  [card] [card]           │  [card] [card] [card] [c]│
│                          │  [card] [card] [card] [c]│
└──────────────────────────┴──────────────────────────┘
```

### 3. Compact Card Design

每个卡片极简化，约 50-60px 高：

```
┌─────────────────────────────────┐
│ [名称 Badge]  [状态]  请求数  成功率  延迟  Token │
└─────────────────────────────────┘
```

- `padding="sm"` `radius="sm"`
- 用 `Badge` 显示状态，`Text` 显示数据
- 用文字百分比 + 颜色替代 `RingProgress`（节省空间）
- 不显示 base_url（已有独立平台页查看）

### 4. All Data Always Visible

- 不做折叠/展开
- 不做分页
- 所有 8 个虚拟大模型和 20 个平台全部展示
